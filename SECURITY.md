# Security Policy

## Supported Versions

We support the latest main branch and the most recent tagged release.

## Reporting a Vulnerability

Please report security issues privately by email to the maintainers or open a GitHub Security Advisory.

- Avoid filing public issues for sensitive reports.
- Include a minimal reproduction and affected versions.
- We aim to respond within 72 hours.

## Handling Secrets

- Do not commit secrets to the repository.
- Use GitHub Secrets for CI and `.env` for local development.
