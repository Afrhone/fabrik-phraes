# Contributing to Fabrik

Thanks for your interest in contributing!

## Getting Started

- Node.js 20+
- `npm install`
- `npm run dev` and open `/api/health`

## Branch & PR

- Create feature branches from `main`
- Keep PRs focused and small
- Link issues and describe changes clearly

## Coding Guidelines

- Prefer minimal dependencies
- Add or update tests when changing behavior
- Update README and manifest for user-facing changes

## Security

- Never commit secrets or tokens
- Use repository secrets or local `.env`

## Release

- CI runs on pushes and PRs
- Tag `vX.Y.Z` to publish a Docker image to GHCR
